What's the point of Figure 7? Use Figure 7 to find the most common design choice within each category for the Visual narrative and Narrative structure (the categories within visual narrative are 'visual structuring', 'highlighting', etc).

The point of Figure 7 is to show categories of narrative visualizations.

Visual Narrative

These are visual devices that assist and facilitate the narrative, they are subdivided into :

  • 1) *Visual structuring*, which are mechanisms that allow the user to keep track of his position within the entire whole.
  • 2) *Highlighting*, which help direct the user's attention to a specific part of the visualization.
  • 3) *Transition guidance* which is used to move from one aspect to another of the visualization without disorienting the user.

    • The most common design choices for each of the categories are the following:

    CATEGORY Visual Structuring Highlighting Transiton Guidance
    MOST COMMON DESIGN Consistant Visual Platform Feature Distinction Animated Transistions

    Narrative Structure

    These refer to non-visual mechanisms that help with telling the story, and are subdivided into:

  • 1) *Ordering*, which refers to the way paths are organized. They could be no indication, allowing random access, there could be a linear path described by the author or it could be left to the user to decide which path to take.
  • 2) *Interativity*, which refers to the ways the user can interact with the story.
  • 3) *Messaging*, the ways comments and observations are presented to the user.

    • The most common design choices for each of the categories are the following:

    CATEGORY Ordering Interactivity Messaging
    MOST COMMON DESIGN User Directed Path Filtering/Selection/Search Captions/Headlines

    Explain in your own words: How is explanatory data analysis different from exploratory data analysis?

    • Exploratory data analysis is when you explore the data. You have a general idea of how the data is gathered and what it aimed to describe even though you can not yet explain how the observed data represents the phenomena being measured or monitored. This involves e.g. outlier detection through visualisation and calculation of general statistics.
    • Explanatory data analysis aimes to explain the data. You have good knowledge of the data and you wish to present it to a given audience. You utilize visualisation methods and compute relevant statistics that you have found to be most descriptive to intuitively get your point accross.

    return pd.to_datetime(datetime.strptime(date.strftime("%Y-%m-%d")+" "+"{:02d}".format(math.floor(time.seconds/3600)) + ":00", "%Y-%m-%d %H:%M"), utc = False)

    From this result, we see that the dataset is imbalanced. Having an imbalanced dataset can lead to misleading results when evaluating the performance of our model. Most machine learning models operate using a loss function which is minimized. In the case of binary classification problems, the loss function is usually the 0-1 loss, which, when used to measure error, it tells us the proportion of miss-classified observations. This, however, can be a misleading measure of how well our model is performing. In our case, for example, if our model were to always predict BURGLARY, it would misclassify $32\%$ of the time, which could be considered quite a good result, but the model would have not learned how to predict future crimes. Creating a balanced dataset is, therefore, particularly useful when we want to have our model learn how to predict even those observations that are more rare.

    A downside of balancing a dataset is that we are altering the distribution of the data. Is what we are interested in is an unsupervised analysis of the data, which aims at describing the distribution of the data, we can see that altering it artificially would make it impossible.

    The balancing needs to be performed by carefully picking random samples from the data available, as to not alter the overall behaviour of the data and introduce unwanted bias in the model.

    Define a function to evaluate the performance of the classifier. And an additional function to plot importance of features.

    In [10]:
    def performance_classifier(clf, dataset, classes, scaler = None):
    from sklearn.metrics import classification_report
    X = dataset.drop(columns="Category").to_numpy()
    if scaler is not None:
        X = scaler.transform(X)
    y_true = dataset["Category"].to_numpy()

    y_hat = clf.predict(X)

    print(classification_report(y_true, y_hat, target_names=classes))
def plot_importance(clf, features, figsize = (6,6)):
    sorted_idx = clf.feature_importances_.argsort()
    feature_names = np.array(features)
    y_ticks = np.arange(0, len(feature_names))
    fig, ax = plt.subplots(figsize = figsize)
    ax.barh(y_ticks,  clf.feature_importances_[sorted_idx])
    ax.set_yticklabels(feature_names[sorted_idx])
    ax.set_yticks(y_ticks)
    ax.set_title("Random Forest Feature Importances (MDI)")
    fig.tight_layout()
    plt.show()

    Comment on the results

    The first result we see is that we have an increase in the performance of our model after we add the weather data. This is the performance on unseen data, so it's significant. This result is indicative of the fact that the additional weather information is useful to better predict what crime will happen in the future. The fact that the performance of prediction in both crimes sees an improvement is due also to the fact that when we can predict the category that is weather-depended better we will, in turn, predict also the other category more reliably (in the label-endoded model, the recall is mostly improved in the weather-dependent Burglary, from 67 to 77, instead of Fraud which goes from 70 to 75, which is what we expect).

    Thanks to the feature importance graphs we can also see that the hour of the week and the temperature are the most important features in determining the category of a crime, followed by humidity, pressure and month of the year. Among the weather classes, we see that to most important one is whether the sky is clear, which makes sense for a crime like burglary.

    We picked features based on the feature importance graph on models which used more complex features and on assumptions on the nature of the crime. We expected that the hour of the week would not be enough to predict crimes, as we saw in previous week that a monthly pattern is also present, i.e. incident numbers were different between summer months and winter months. Additionally, we removed wind-direction as we deemed it not influencing the crime rate.
